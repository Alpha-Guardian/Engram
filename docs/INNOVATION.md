# Innovation Object

This repo is not positioned as a generic PEFT release and not as a pure deployment showcase.

## What Is New Here

The public innovation object is a coupled regime:

- `frozen authoritative parent`
- `residual-family local repair`
- `route-gated trunk recurrent activation`
- `coupled promotion contract`
- `derived bounded board artifact`

The claim is that these pieces are meaningful as one accepted object. A repair does not count merely because it improves one slice. It must remain anchored to the frozen parent, survive the promotion predicate, and still be crystallizable into a bounded deployment artifact.

## Why That Is Different From Simpler Framings

This is narrower than:

- "we invented a new PEFT primitive"
- "we quantized a model to MCU"
- "we added routing and got a better score"

And it is broader than:

- "we have one helpful local patch"
- "we have one board benchmark demo"

The public bundle ties mechanism, evidence, and deployment boundary together.

## What Remains Private

This public framing does not disclose the main training moat.

Still private:

- full family-mining and candidate search
- support and carrier discovery logic
- rejected frontier and failure cards
- full route token inventory and source tracing
- curriculum generation and full training pipeline

That separation is intentional. The repo aims to prove the accepted object without publishing the private asset layer that produced it.
